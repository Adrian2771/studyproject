Longest substring in alphabetical order

The following exercise is part of a mandatory assignment from a Computer Science course delivered by the edX platform. The original requirements are for implementation in Python, but the solution can be written in any programming language, using loops and decision branches.

In short, given a certain string of characters s, the student is asked to write a program that prints the longest substring of s in which the letters occur in alphabetical order. For example, if s = 'azcbobobegghakl', then the program should print "Longest substring in alphabetical order is: beggh"

In the case of ties, the first substring should be printed. For example, if s = 'abcbcd', then the program should print "Longest substring in alphabetical order is: abc". The following implementation does a little more. After entering a string of characters in field below, you have three options.

The Fibonacci sequence, recursive and iterative

The Fibonacci numbers are well explained over the internet, thus they won't be analysed here. Their role is important in understanding some concepts such as iterative calculus and recursive function calls. In short, the Fibonacci numbers can be generated with the following recursion formula: Fn = Fn-1 + Fn-2. The first Fibonacci number is 0 and the second Fibonacci number is 1. So, the third number in sequence will be (1 + 0 =) 1, the fourth number, 2, the fifth, 3 etc.

The iterative method to determine the n-th Fibonacci number requires a loop in which

i) res = a + b; ii) b = a; iii) a = res,

for each iteration from 3 to n, including n. res, a and b are variables of integer type. The initial values for a and b are 0 and 1, the first two Fibonacci numbers.

The recursive method has a minor advantage over the iterative method: it doesn't need a loop to perform the calculus. This, however, comes with a cost: the execution time for a small piece of code increases rapidly to seconds or tens of seconds, which in programmers' world is considered an undesirable, ineffective behavior. The reason behind this ineffectiveness is the so-called binary tree. Let's consider the following recursive implementation:

function F(z) {
if(1 === z || 2 === z) return z - 1;
return F(z - 1) + F(z - 2);
}

If we want to determine the 4th Fibonacci number, we'll call the function F with the argument 4. The function will have to return F(3) + F(2). Each of the two members of the expression is a function call made by the function itself, with a specific argument. F(2) evaluates to 1 but F(3) leads to other two function calls. When called with the argument 3, the function has to return F(2) + F(1), which evaluates to 1 and 0 respectively. We can count easily 5 function calls. For F(5) we get 9 function calls, for F(6), 15 calls etc. The recursive method is not recommended in such cases: the execution time grows at an exponential rate.

The Binet formula is another method to determine the Fibonacci numbers.

F(n) = ( (1 + √ 5)n-1 - (1 - √ 5)n-1 ) / (2n * √ 5)

It requires a 'floating point to integer' conversion (a ceil rounding), since the result is a real number. The Binet method and the iterative method have reasonable execution times.
